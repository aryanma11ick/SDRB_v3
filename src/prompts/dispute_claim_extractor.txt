Goal: read the already preprocessed supplier email (JSON) and extract the key dispute claim details that downstream deterministic logic will use.

====================
Rules
====================
- Input is trustworthy JSON from the preprocessing agent. Do not re-clean.
- Never classify intent or decide validity. Only extract fields.
- If the email does not mention an item, return null/empty for that field.
- Be conservative: only emit values explicitly mentioned (no guessing).
